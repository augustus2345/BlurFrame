## 1. 概述（Overview）

### 1.1 背景

个人用户在手机里会积累大量照片，常见的痛点：

- **没有"仪式感"**：相机直出的照片缺少修饰，社交分享时不够"好看"。
- **打标签依赖云端**：Lightroom 等专业工具用得顺，但需要付费/同步/登录。
- **整理靠手动新建文件夹**：系统相册没有"主题相册"概念，找回忆成本高。
- **清理照片反人性**：一张张看、一张张删，挫败感强。

### 1.2 目标

打造一个**本地优先、零账号、零网络**的照片管理 App，让个人用户能够：

1. 给照片套上好看的**模版**（内置 + 用户自定义模版）。
2. 把零散的照片**组合成影集**。
3. 像 Lightroom 一样**打标签、加星级、搜索**。
4. 用**手势**（上滑 / 左右滑）快速删除照片。

### 1.3 成功指标

- 单帧模版合成 ≤ 1.5s（1080p 设备）。
- 1000 张照片下相册网格首屏 ≤ 300ms。
- 7 日留存 ≥ 35%（自用验证）。

## 5. 非功能需求（Non-Functional Requirements）

| 类别 | 要求 |
| --- | --- |

## 6. 架构与实现决策（Implementation Decisions）

> 这一节是开发者视角的关键决策；具体代码会变，但决策要稳定。

### 6.1 模块划分

按 feature 划分，目录结构沿用 README 中已声明的骨架：

```
lib/
├── core/         # 跨 feature：theme、router、constants、widgets
├── shared/       # services（Hive、Settings）+ 通用 widgets
└── features/
    ├── photos/   # 相册主屏、详情、删除 tab
    ├── frames/   # 模版与编辑器（独立 tab）
    ├── albums/   # 影集
    ├── tags/     # 标签
    ├── search/   # 搜索二级页（push）
    └── settings/ # 设置
```

每个 feature 内部分 `data/`（models / datasources / repositories）和 `presentation/`（providers / screens / widgets）。

### 6.2 数据模型

- **`Photo`**：纯数据模型，承载"用户态"——`id`（assetId）、`tags`、`starRating`（0–5）、`frameTemplateId`、`notes`、`addedAt`。**不持有图片字节**。
- **`FrameTemplate`**：模版元数据。`layers: List<FrameLayer>`。`FrameLayer` 是 sum type（discriminated union，3 种）：
  - `BlurBorderLayer`：模糊边框
  - `TextWatermarkLayer`：底部水印（自定义文案 + 可选 EXIF 字段）
  - `ColorStripeLayer`：颜色标签条
  - 运行时统计 `usageCount`（写入 Hive，每次"应用模版"+1）
- **`Album`**：影集。`photos: List<String>`（assetId 列表）、`name`、`coverAssetId`、`layout`。
- **`Tag`**：标签。`id`、`name`、`color`。
- **`SearchFilter`**：当前过滤条件。`tagIds`、`minStarRating`、`dateRange`、`albumId`、`framedState`（all / framed / unframed）。

### 6.3 状态管理

- 用 `flutter_riverpod`，**Provider 按 feature 域划分**，不按页面划分。
- 全局状态（当前选中模版、删除进度、过滤条件、多选集合）放在 provider，单一数据源。
- 页面级短暂态用 `StatefulWidget` + `setState`，不污染 provider。

### 6.4 持久化

- 全部元数据走 `hive`。每个 model 用 `@HiveType` + `@HiveField` 标注。
- 模板/标签/影集/搜索条件用独立 box。
- 照片元数据（tags、starRating、frameTemplateId）以 assetId 为 key，存到 `photoMeta` box。
- 原图字节**不复制**，全部留在系统相册。

### 6.5 渲染

- 模版渲染 = `image` 包 + 自定义 Painter，按 z-order 合成。
- 渲染统一走 `FrameRenderer.render(Photo, FrameTemplate) -> Future<Uint8List>`。
- 渲染在 `compute` 隔离中跑，避免阻塞 UI。
- 导出走 `gal` 插件写入系统相册。

### 6.6 路由

- `go_router` + `ShellRoute`（**底部 5 tab**：相册 / 影集 / 模版 / 删除 / 设置）。
- **二级页**（push，不进 Shell）：
  - `/photo/:assetId` — 照片详情
  - `/albums/:id` — 影集详情
  - `/search` — 搜索二级页（入口在相册 tab 顶部搜索栏）
  - `/frames/editor` — 模板编辑器（新建 / 编辑）
  - `/frames/:id` — 模版详情 / 预览

### 6.7 防竞态

- 异步流程（渲染、导出、删除）启动时分配一个 `sessionId`/`cancelToken`，UI 状态变更前校验，过期丢弃。

### 6.8 国际化

- `flutter_localizations` + `intl`，`lib/l10n/app_zh.arb`、`app_en.arb`。**首期仅中文，框架预留**。

### 6.9 UI 主题

- Material 3，`AppColors` 集中管理：`surface`、`onSurface`、`accent`（暖色）、`muted`。
- 字体：Google Fonts `Inter`（英文）+ 系统字体（中文）。
- 暗色模式完整支持，照片在两种模式下都保持真实色彩（不做额外滤镜）。

### 6.10 错误与空态

- 所有异步路径显式处理 4 态：loading / success / error / empty。
- 错误用 snackbar（带"重试"），空态用 `EmptyState` 通用组件。
- 删除类破坏性操作走底部确认 sheet。

## 7. 测试决策（Testing Decisions）

> 核心：测"外部行为"（public API、用户可见的副作用），不测 private 实现细节。

### 7.1 测试原则

- **优先测状态流转和业务逻辑**（CLAUDE.md 2.6.28），不是 widget 表面。
- 一份测试只测一件事；测试名即文档。
- 不测 `flutter_riverpod` 自身、不测 Dart 标准库。
- 第三方库（`photo_manager`、`gal`）用 `mocktail` 替身注入。

### 7.2 测试矩阵

| 层级 | 范围 | 工具 | 数量目标 |
| --- | --- | --- | --- |
| Unit | 仓储层逻辑、FrameRenderer 纯合成逻辑、SearchFilter 匹配规则、tag color 工具 | `flutter_test` | 覆盖核心模型 100% 公共方法 |
| Widget | 关键 widget 的存在性、回调触发、Loading/Empty/Error 态切换 | `flutter_test` + `ProviderScope.overrides` | 关键 screen × 1 |
| Integration | "打开 App → 选照片 → 套模板 → 导出"主流程 | `integration_test` | 至少 1 条主链路 |

### 7.3 关键测试用例（必须覆盖）

1. **`FrameRenderer`**：纯输入 → 纯输出（输入 assetId/bytes + 模板 → 期望 bytes 长度 / 尺寸），跑在 `compute` 之外。
2. **`PhotoRepository`**：用 `photo_manager` mock，验证 `loadAll`/`getExif` 异常传播正确。
3. **`SearchFilter.matches(photo)`**：标签组合、星级、日期范围、模版状态四维度交叉用例。
4. **`AlbumRepository.addPhotos` / `removePhotos`**：顺序保持、重复去重、跨相册引用一致。
5. **`TagRepository.delete`**：被引用的 tag 不能裸删，必须先解绑或提示。
6. **删除 tab 状态机**：↑ 删 / ← 上一张 / → 下一张 / 顶栏 `⋯` 操作 四条路径。
7. **多选 → 批量操作**：选择集合的变化与 provider 单一数据源一致。
8. **Widget**：空态组件的文案、loading 的 indicator、错误 snackbar 触发。
9. **Integration**：打开 → 授权 → 看到网格 → 进详情 → 套模板 → 导出。

### 7.4 测试 mock 策略

- `PhotoRepository` 走接口（`abstract class`），UI / Provider 拿接口不拿实现。
- 测试用 `MockPhotoRepository`（mocktail）注入。
- 渲染用本地 fixture 图片（`assets/test/photo.jpg`），不依赖系统相册。

## 8. 里程碑规划（Milestones）

> 每个里程碑结束都要：✅ 跑通主链路 / ✅ 测试通过 / ✅ TASKS.md 已更新。

| Milestone | 内容 | 验收 | 估时 |

## 10. 用户故事（User Stories）

> 按用户视角写，**给开发、测试、设计共同参考**。每条 ID 映射回 §4 的功能 ID。

### 照片库
1. 作为用户，我想在第一次打开 App 时看到清晰的授权引导，这样我能放心授予相册权限。
2. 作为用户，我想看到一个清爽的网格布局，这样我能快速浏览大量照片。
3. 作为用户，我想双指缩放查看照片细节，这样我能确认对焦是否清晰。
4. 作为用户，我想长按照片进入多选模式，这样我能批量处理。
5. 作为用户，我想看到照片的 EXIF 信息（机型、时间、ISO），这样我能回忆拍摄场景。
6. 作为用户，我想在照片详情页一键跳到"模版"功能，这样我能快速美化。
7. 作为用户，我想要暗色模式，这样夜间看照片不刺眼。

### 模版
8. 作为用户，我想看到至少 2 套开箱即用的模版（**极简** / **杂志**），这样我不用从零开始。
9. 作为用户，我想给照片加底部水印（自定义文案 + 可选 EXIF），这样照片更"专业"。
10. 作为用户，我想给照片加颜色标签条（基于照片主色或指定色），这样照片更有"杂志感"。
11. 作为用户，我想把照片边缘做模糊处理，这样主体更突出。
12. 作为用户，我想新建一个模版，并保存下来反复使用。
13. 作为用户，我想复制内置模版作为起点，再改造成自己的版本。
14. 作为用户，我想在编辑器里实时看到套用效果，这样我能即时调整。
15. 作为用户，我想把套好模版的照片导出到系统相册，这样我能分享到朋友圈。
16. 作为用户，我希望导出的图清晰，且不会卡顿（≤ 1.5s）。
17. 作为用户，我想在**模版 tab**浏览所有模版，看到使用次数和"自带"标记。

### 影集
18. 作为用户，我想从多张照片一键创建"影集"，这样我能按主题整理回忆。
19. 作为用户，我想给影集起名 + 选封面，这样我一眼能识别。
20. 作为用户，我想在影集详情里用多种版式浏览，这样视觉更丰富。
21. 作为用户，我想拖拽重排影集内照片的顺序，这样故事有节奏。
22. 作为用户，我想更换影集封面，这样我能挑出最满意的一张。

### 标签 + 搜索
23. 作为用户，我想给照片加多个标签，这样我能多维度分类。
24. 作为用户，我想在专门的标签管理页看到所有标签和数量，这样我能掌握全貌。
25. 作为用户，我想给照片打 0–5 星，作为与标签并列的快速分类维度。
26. 作为用户，我想在**相册 tab 顶部搜索栏**输入关键字，进入搜索二级页（push），按**标签 / 星级 / 时间 / 影集**多维过滤，这样我能精准定位。
27. 作为用户，我想把多个过滤条件组合（标签 + 星级 + 时间 + 影集），这样搜索更准。
28. 作为用户，我想在搜索结果中继续打标签 / 加星 / 删除，这样减少来回切换。

### 批量 + 删除 tab
29. 作为用户，我想一次给多张照片套同一个模版，省时省力。
30. 作为用户，我想批量打标签 / 批量加星，不想一张张点。
31. 作为用户，我想在删除前有确认提示，避免误删。
32. 作为用户，我想切换到**删除 tab**后单图全屏浏览，**上滑**快速删除，**左/右滑**切换上下张，这样我能在 5 分钟内清理几十张。
33. 作为用户，我希望误删后能撤销（5 秒内），这样不焦虑。
34. 作为用户，我想看到当前是 `N / M` 第几张，这样我知道进度。
35. 作为用户，我想在删除 tab 顶栏右上 `⋯` 切换过滤或批量操作。

### 设置
36. 作为用户，我想切换浅色/深色/跟随系统主题。
37. 作为用户，我想切换 App 语言（简体中文 / English）。
38. 作为用户，我想查看存储空间占用（缓存 + 元数据）。
39. 作为用户，我想从其他 App 导入 / 导出照片元数据（JSON）。
40. 作为用户，我想看到 App 版本和关于信息。

## 11. 风险与开放问题（Risks & Open Questions）

| # | 风险/问题 | 应对 |
| --- | --- | --- |
| R1 | `photo_manager` 在部分 Android 厂商上不稳定 | 适配层加 try/catch，UI 给清晰错误提示 |
| R2 | 大图渲染内存峰值 | 缩略图/详情/导出 三档尺寸分离，避免同一张原图多次解码 |
| R3 | EXIF 在转码后丢失 | 渲染时把 EXIF 重新写回（`piexif`/`exif` 包） |
| R4 | 删除 tab 手势与系统返回冲突 | 用 `GestureDetector` + 明确方向判定（上 / 左 / 右），禁掉删除 tab 的页面返回 |
| R5 | 模版图层叠加性能 | 渲染走 `compute`，UI 端只显示进度 |
| R6 | 用户对"好看"的定义主观 | 内置 2 套先验，再让用户自定义 |
